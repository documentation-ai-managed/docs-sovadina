## Overview

Sovadina provides secure authentication tailored for HealthTech applications. You authenticate using API keys for simple access, OAuth 2.0 for third-party integrations like FHIR servers, and role-based access control (RBAC) for team permissions. These methods ensure compliance with HIPAA and GDPR while handling sensitive patient data.

## API Keys

Generate API keys from your Sovadina dashboard at `https://dashboard.example.com/settings/api-keys`. Each key supports scoped permissions for endpoints like patient records or FHIR resources.

<Callout kind="alert">
  Store `YOUR_API_KEY` securely. Never commit it to version control or expose it in client-side code.
</Callout>

### Setup Steps

<Steps>
  <Step title="Create Key" icon="key">
    Log in to `https://dashboard.example.com` and navigate to API Keys. Click "Generate New Key".
  </Step>
  <Step title="Set Permissions" icon="settings">
    Select scopes: `read:patients`, `write:fhir`.
  </Step>
  <Step title="Use in Requests" icon="code">
    Add the key to your `Authorization` header.
  </Step>
</Steps>

<ParamField header="Authorization" param-type="string" required="true">
  Bearer token format: `Bearer YOUR_API_KEY`.
</ParamField>

<CodeGroup tabs="JavaScript,Python,cURL">
  ```javascript
  const response = await fetch('https://api.example.com/v1/patients', {
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json'
    }
  });
  ```
  ```python
  import requests
  response = requests.get(
    'https://api.example.com/v1/patients',
    headers={'Authorization': 'Bearer YOUR_API_KEY'}
  )
  ```
  ```bash
  curl -H "Authorization: Bearer YOUR_API_KEY" \
       https://api.example.com/v1/patients
  ```
</CodeGroup>

## OAuth 2.0 Flows

Use OAuth for integrations with EHR systems or FHIR-compliant services. Sovadina supports Authorization Code and Client Credentials flows.

<Tabs>
  <Tab title="Authorization Code" icon="login">
    For user-interactive apps.

    <Steps>
      <Step title="Redirect to Auth" icon="external-link">
        Send user to `https://auth.example.com/oauth/authorize?client_id=YOUR_CLIENT_ID&redirect_uri=YOUR_REDIRECT&scope=read:patients`.
      </Step>
      <Step title="Exchange Code" icon="key">
        Use code for token: POST to `/oauth/token`.
      </Step>
    </Steps>

    ```javascript
    const tokenResponse = await fetch('https://auth.example.com/oauth/token', {
      method: 'POST',
      headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
      body: new URLSearchParams({
        grant_type: 'authorization_code',
        code: 'AUTH_CODE',
        client_id: 'YOUR_CLIENT_ID',
        redirect_uri: 'YOUR_REDIRECT'
      })
    });
    ```
  </Tab>
  <Tab title="Client Credentials" icon="server">
    For machine-to-machine.

    <ParamField header="Authorization" param-type="string" required="true">
      Basic auth: `Basic BASE64(CLIENT_ID:CLIENT_SECRET)`.
    </ParamField>

    ```javascript
    const token = await fetch('https://auth.example.com/oauth/token', {
      method: 'POST',
      headers: {
        'Authorization': 'Basic ' + btoa('YOUR_CLIENT_ID:YOUR_CLIENT_SECRET'),
        'Content-Type': 'application/x-www-form-urlencoded'
      },
      body: 'grant_type=client_credentials&scope=write:fhir'
    });
    ```
  </Tab>
</Tabs>

## Role-Based Permissions

Assign roles like `admin`, `clinician`, or `viewer` to team members. Roles control access to endpoints and dashboards.

| Role      | Permissions                  | Use Case                  |
|-----------|------------------------------|---------------------------|
| Admin    | All endpoints, key management| Project owners            |
| Clinician| Read/write patients, FHIR   | Medical staff             |
| Viewer   | Read-only access             | Auditors                  |

<Expandable title="Advanced RBAC Setup" default-open="false">
  Configure custom roles via API:

  ```javascript
  await fetch('https://api.example.com/v1/roles', {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      name: 'radiologist',
      permissions: ['read:images', 'write:reports']
    })
  });
  ```
</Expandable>

## Compliance and Best Practices

Sovadina auth complies with HIPAA and GDPR. Encrypt tokens in transit (HTTPS only) and rotate keys every 90 days.

<Callout kind="tip">
  Enable audit logs in dashboard for all auth events. Review access monthly.
</Callout>
